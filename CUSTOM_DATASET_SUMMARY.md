# Custom Dataset Setup - Summary

This document summarizes the changes made to support your custom datasets (IXI T1/T2, MOOD, and BraTS).

## What Was Added

### 1. MOOD Dataset Support
- **File**: `src/datamodules/Datamodules_eval.py`
  - Added `MOOD` class following the same pattern as BraTS and MSLUB
  - Handles optional segmentation files (not all MOOD images may have ground truth)

- **File**: `configs/datamodule/IXI.yaml`
  - Added MOOD dataset paths configuration
  - Added MOOD to the list of test sets to evaluate

### 2. Documentation

Created three comprehensive guides:

- **`QUICKSTART.md`**: Fast-track guide to get you up and running quickly
  - Step-by-step commands
  - Common use cases
  - Troubleshooting tips

- **`DATASET_SETUP_GUIDE.md`**: Detailed documentation
  - Complete directory structure requirements
  - CSV file format specifications
  - Image requirements and preprocessing
  - Advanced configuration options
  - Common issues and solutions

- **`scripts/README.md`**: Script usage documentation
  - Detailed explanation of each utility script
  - Examples and parameters
  - Troubleshooting specific to data preparation

### 3. Utility Scripts

Created two Python scripts to automate setup:

- **`scripts/generate_csv_splits.py`**
  - Automatically generates CSV split files from your data
  - Scans directories and matches images with masks/segmentations
  - Handles IXI, BraTS, and MOOD datasets
  - Includes verification functionality

- **`scripts/verify_dataset.py`**
  - Comprehensive pre-training verification
  - Checks directory structure, CSV files, and file existence
  - Optional dimension checking
  - Clear pass/fail reporting

### 4. CSV Templates

Created example templates in `Data/splits/templates/`:
- `IXI_train_template.csv`: Format for IXI training data
- `MOOD_test_template.csv`: Format for MOOD test data
- `MOOD_val_template.csv`: Format for MOOD validation data

## How to Use Your Datasets

### Quick Start (TL;DR)

```bash
# 1. Setup environment
conda activate cddpm-uad
export DATA_DIR=/path/to/your/data

# 2. Organize your data following the structure in DATASET_SETUP_GUIDE.md

# 3. Generate CSV files
python scripts/generate_csv_splits.py --data_dir $DATA_DIR --dataset ixi --modality t1 --split train --fold 0
python scripts/generate_csv_splits.py --data_dir $DATA_DIR --dataset brats --modality t2 --split test
python scripts/generate_csv_splits.py --data_dir $DATA_DIR --dataset mood --modality t2 --split test

# 4. Verify setup
python scripts/verify_dataset.py --data_dir $DATA_DIR

# 5. Train
python run.py experiment=cDDPM/Spark_2D_pretrain datamodule.cfg.mode=t1
python run.py experiment=cDDPM/DDPM_cond_spark_2D encoder_path=<path> datamodule.cfg.mode=t1
```

### File Naming Convention

- Images: `<ID>_t1.nii.gz` or `<ID>_t2.nii.gz`
- Masks: `<ID>_mask.nii.gz`
- Segmentations: `<ID>_seg.nii.gz`

Example:
- `IXI001-Guys-0740_t1.nii.gz`
- `IXI001-Guys-0740_mask.nii.gz`
- `BraTS2021_00001_t2.nii.gz`
- `BraTS2021_00001_mask.nii.gz`
- `BraTS2021_00001_seg.nii.gz`

## Training Workflows

### Workflow 1: Train on IXI T1, Test on BraTS and MOOD

```bash
# Pretrain encoder on IXI T1
python run.py experiment=cDDPM/Spark_2D_pretrain datamodule.cfg.mode=t1

# Train full model
python run.py experiment=cDDPM/DDPM_cond_spark_2D \
    encoder_path=<path_from_pretraining> \
    datamodule.cfg.mode=t1

# Model will automatically evaluate on BraTS, MOOD, and IXI test sets
```

### Workflow 2: Train on IXI T2, Test on BraTS and MOOD

```bash
# Pretrain encoder on IXI T2
python run.py experiment=cDDPM/Spark_2D_pretrain datamodule.cfg.mode=t2

# Train full model
python run.py experiment=cDDPM/DDPM_cond_spark_2D \
    encoder_path=<path_from_pretraining> \
    datamodule.cfg.mode=t2
```

### Workflow 3: Cross-Validation on IXI

```bash
# Run all 5 folds
for fold in {0..4}; do
    # Pretrain
    python run.py experiment=cDDPM/Spark_2D_pretrain \
        datamodule.fold=$fold \
        datamodule.cfg.mode=t1
    
    # Train
    python run.py experiment=cDDPM/DDPM_cond_spark_2D \
        datamodule.fold=$fold \
        datamodule.cfg.mode=t1 \
        encoder_path=<path>
done
```

## What Datasets Are Evaluated

During training, the model automatically evaluates on:
- тЬУ **BraTS dataset** (brain tumors)
- тЬУ **MOOD dataset** (various anomalies)
- тЬУ **IXI test set** (healthy controls as baseline)
- тЬУ **MSLUB** (if you have it - multiple sclerosis lesions)

You can customize which datasets to evaluate by editing `testsets` in `configs/datamodule/IXI.yaml`.

## Key Configuration Files

- **`configs/datamodule/IXI.yaml`**: Dataset paths and parameters
  - Modify `mode` to switch between T1 and T2
  - Adjust `batch_size`, `num_workers` for your hardware
  - Specify which test sets to evaluate

- **`pc_environment.env`**: Paths configuration
  - Set `DATA_DIR` to your data location
  - Set `LOG_DIR` for outputs and checkpoints

## Expected Outputs

Training produces:
- **Checkpoints**: `$LOG_DIR/logs/train/runs/<timestamp>/checkpoints/`
- **Logs**: Training curves, metrics
- **Evaluation results**: Dice scores, AUROC, AUPRC for each test set
- **Visualizations**: Sample reconstructions and anomaly maps

## Common Issues

| Issue | Solution |
|-------|----------|
| Out of memory | Reduce `batch_size` in config |
| Missing files | Run `verify_dataset.py` to check |
| Wrong paths | Ensure CSV paths are relative to DATA_DIR |
| Dimension mismatch | Use preprocessing/resample.py |
| Import errors | Activate conda environment |
